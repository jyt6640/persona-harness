# Code of Conduct

## Our standard

Persona Harness is a small, technical, evidence-first project. The working norm
is simple: **be respectful, be precise, and back claims with evidence.**

Expected:

- Assume good faith. Critique ideas and code, not people.
- Prefer a narrow true statement over a broad attractive one — in reviews too.
- Disagree with data or reproducible reasoning, not volume.

Not tolerated:

- Harassment, personal attacks, discriminatory or demeaning language.
- Sustained disruption, bad-faith argument, or dismissing others' work without
  substance.

## Scope

This applies to all project spaces: issues, pull requests, discussions, and any
project communication channel.

## Enforcement

Report conduct concerns privately to the maintainer via the email on the npm
package page, or through GitHub's report feature. The maintainer may warn, edit
or remove contributions, or block accounts as needed. This is a single-maintainer
project; decisions are made in good faith and are final.

This is adapted from the spirit of the
[Contributor Covenant](https://www.contributor-covenant.org), kept short on
purpose for a project of this size.
