## Finding

Build-line smoke: PASS.

The tightened guidance appears to have reduced the previously repeated `bootJar` workaround in this clean run:

- OpenCode selected a compatible Spring Boot / Gradle / Java line.
- `bootJar` stayed enabled.
- `gradle test` passed.
- `gradle build` passed with `:bootJar`.
- `gradle bootRun` and basic HTTP smoke passed.

## Limitations

- Single clean smoke run.
- Not an enforcement gate.
- Not generated app product-quality certification.
- Not proof that future models will never disable `bootJar`.
- In-memory storage was allowed by this fixture, so persistence durability was not evaluated.

## Decision

Keep the tightened Gradle/Spring build guidance.

Do not start another broad package-shape A/B loop from this result. Reopen only if a future clean run regresses on executable Spring Boot build-line behavior.
