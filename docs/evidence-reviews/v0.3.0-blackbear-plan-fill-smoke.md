# v0.3.0 Blackbear Plan Fill Smoke

## Goal

Verify that a clean Java/Spring backend project can run:

```bash
npx ph init
npx ph intake
npx ph plan
```

Then verify OpenCode can read `README.md` and `.persona/workflow/plan.md`, complete the backend architecture/technology plan, and stop before implementation.

## Setup

- Date: 2026-06-20
- Persona Harness package: local tarball from current worktree
- Temp project: `/tmp/persona-v030-blackbear-plan-smoke`
- Model: `openai/gpt-5.4-mini-fast`
- Prompt:

```text
README.md와 .persona/workflow/plan.md를 끝까지 읽어라. 코드는 만들거나 수정하지 말고, .persona/workflow/plan.md의 Architecture / Technology Plan 섹션을 backend 계획으로 구체화해라. 구현은 시작하지 말고 마지막 줄에 IMPLEMENTATION_NOT_STARTED 라고 써라.
```

## Result

PASS.

OpenCode:

- read `README.md`;
- found `.persona/workflow/plan.md` after checking hidden `.persona` paths through shell;
- read `.persona/workflow/plan.md`;
- read `.persona/project-profile.jsonc`;
- updated only `.persona/workflow/plan.md`;
- did not create `src/`, `build.gradle`, `settings.gradle`, or `pom.xml`;
- ended with `IMPLEMENTATION_NOT_STARTED`.

## Plan Output

The completed plan included:

- fixed use cases: equipment registration, list, member registration, rental, return;
- rental rule: available stock and not currently rented;
- return rule: only the renter can return;
- distinct domain exceptions for missing equipment/member/rental;
- Java 17+, Spring Boot, Spring Web, Spring Validation, Spring JDBC, Flyway, Gradle;
- relational database plus `JdbcTemplate` repositories;
- Flyway migration;
- domain-first package structure;
- `presentation -> application -> domain -> infrastructure` flow;
- strict controller DTO and application command/result DTO boundary;
- equipment/member/rental domain responsibility split;
- application service orchestration-only;
- repository/store ownership for state and ID generation;
- common JSON error response.

## Known Gaps

- `ph` bin conflict was observed with an incomplete tarball produced by an earlier concurrent build/pack attempt. Sequential `npm run build && npm pack` fixed the tarball and linked `node_modules/.bin/ph -> ../persona-harness/dist/cli/index.js`.
- OpenCode glob did not find hidden `.persona` paths directly, but shell `ls -la .persona` followed by direct reads worked.
- This smoke does not verify implementation quality.
- This smoke does not implement `Charles`, `jaeki`, or `roach`.
- This smoke does not prove autonomous planning; it verifies a local plan-fill artifact path.

## Decision

`ph plan` is useful enough to keep as the first blackbear workflow artifact.

Next candidate:

- run a plan-based implementation smoke where OpenCode reads the completed `.persona/workflow/plan.md`, implements the backend, and then records implementation/review evidence separately.

Do not add frontend/infra intake or autonomous agent roles before this backend plan-based implementation smoke.
