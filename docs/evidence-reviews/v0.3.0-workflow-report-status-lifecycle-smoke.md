# v0.3.0 Workflow Report Status Lifecycle Smoke

## Goal

Verify that workflow reports start as `Status: template` and can be explicitly marked as `Status: filled` after they are filled.

This addresses the live HTTP QA OpenCode smoke gap where a useful filled implementation report still said `Status: template`.

## Command Surface

```bash
ph plan --report-filled implementation
ph plan --report-filled review
```

These commands update only workflow report metadata. They do not update plan acceptance, run tests, enforce rules, or certify generated app quality.

## Test Result

Targeted test:

```bash
npx vitest run tests/persona-harness-plan.test.ts
```

Result: PASS, 8 tests.

The added coverage verifies:

- `ph plan` creates implementation/review reports with `Status: template`;
- generated report templates tell the actor to run the matching `ph plan --report-filled ...` command after filling;
- `ph plan --report-filled implementation` marks only `.persona/workflow/implementation-report.md` as `filled`;
- `ph plan --report-filled review` marks only `.persona/workflow/review-report.md` as `filled`;
- `.persona/workflow/plan.md` keeps its plan acceptance status;
- missing reports fail with a clear message.

## CLI Smoke

Clean temporary project:

```bash
tmp=$(mktemp -d /tmp/persona-report-status-XXXXXX)
printf '# Report Status Smoke\n' > "$tmp/README.md"
cd "$tmp"
node /Users/yongtae/Desktop/persona-harness/dist/cli/index.js plan --force
printf 'implementation evidence\n' >> .persona/workflow/implementation-report.md
printf 'review evidence\n' >> .persona/workflow/review-report.md
node /Users/yongtae/Desktop/persona-harness/dist/cli/index.js plan --report-filled implementation
node /Users/yongtae/Desktop/persona-harness/dist/cli/index.js plan --report-filled review
rg -n '^Status:' .persona/workflow/plan.md .persona/workflow/implementation-report.md .persona/workflow/review-report.md
```

Observed status lines:

```text
.persona/workflow/implementation-report.md:4:Status: filled
.persona/workflow/review-report.md:4:Status: filled
.persona/workflow/plan.md:4:Status: draft
```

## Finding

PASS.

The minimal lifecycle is now observable:

```text
template -> filled
```

## Known Limits

- `filled` means the report file was explicitly marked after being filled. It does not prove the report is correct.
- There is no `completed` state yet.
- There is no enforcement gate requiring reports to be filled before `ph history`.
- There is no autonomous `jaeki` or `roach` agent that runs the command automatically.

## Decision

Keep the lifecycle narrow. `filled` is enough for v0.3.0 workflow evidence hygiene.
