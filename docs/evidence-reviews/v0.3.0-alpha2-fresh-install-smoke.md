# v0.3.0-alpha.2 Fresh Install Smoke

## Goal

Verify the external tester path for the current alpha package:

```bash
npm install -D persona-harness@alpha
npx ph init
npx ph intake --interactive
npx ph policy init
npx ph plan
opencode . --model openai/gpt-5.4-mini-fast --prompt "$(npx ph plan --prompt)"
```

This smoke checks install and workflow behavior only. It does not certify generated Spring application quality.

## Initial Package State

- Published `persona-harness@0.3.0-alpha.1` exists on npm.
- Before this loop, npm dist-tags were:
  - `alpha`: `0.3.0-alpha.1`
  - `latest`: `0.3.0-alpha.0`
- Attempting to move `latest` failed with npm `EOTP`; npm requires a one-time password for write actions.
- `0.3.0-alpha.1` fresh install exposed stale `ph init` guidance that still printed implementation-first OpenCode instructions.
- Therefore `0.3.0-alpha.1` should not be promoted as the tester default.

Command:

```bash

```

Result: PASS with caveat.

Observed through the TUI:

- OpenCode accepted the generated plan-only prompt.
- The visible prompt contained README/profile/policies/workflow plan references.
- The visible prompt included `구현하지 말고` and `npx ph bearshell`.
- OpenCode reported that `.persona/workflow/plan.md` was completed.
- OpenCode reported that implementation did not start.
- Filesystem check found no `src/main/java` implementation files.
- `.persona/evidence/phase0` contained README bootstrap evidence.

Caveat:

- The TUI output is escape-sequence heavy and was not useful as a durable text log.
- The durable evidence for this smoke is therefore the generated plan file, absence of implementation files, and `.persona/evidence` JSON.
