# v0.3.0 Profile Summary Injection Smoke

## Goal

Verify whether a filled `.persona/project-profile.jsonc` can influence OpenCode's first architecture/technology plan through Persona Harness injection without explicitly telling OpenCode to read the profile file.

This smoke checks profile summary injection as planning context. It does not certify generated app quality and does not test full implementation.

## Setup

- Persona Harness repo: `/Users/yongtae/Desktop/persona-harness`
- Packaged artifact: `/tmp/persona-harness-0.2.1.tgz`
- Clean smoke project: `/tmp/persona-v030-profile-summary-smoke`

## OpenCode Prompt

The prompt intentionally did not mention `.persona/project-profile.jsonc`.

```text
README.md를 끝까지 읽어라. 코드는 만들거나 수정하지 말고, 구현 전에 필요한 architecture/technology plan만 한국어로 제안해라. 마지막 줄에 IMPLEMENTATION_NOT_STARTED 라고 써라.
```

## Result

PASS.

OpenCode read only `README.md` from the prompt, then produced a planning-only answer that reflected profile-specific values:

- Spring Boot + Gradle backend;
- database storage;
- `JdbcTemplate`;
- `Flyway`;
- strict DTO separation;
- `domain-first` package structure;
- no feature/module wrapper package;
- application service orchestration without Service-owned storage or id sequence.

The run ended with:

```text
IMPLEMENTATION_NOT_STARTED
```

No `src/` implementation files were generated.

## Finding

The backend profile summary injection works as a planning surface.

This is stronger than the previous explicit-profile-read smoke because the user prompt did not name the profile file. The result suggests the profile summary can influence the first plan through injection alone.

## Caveats

- This is one clean smoke, not a statistical guarantee.
- It verifies plan influence, not full generated app quality.
- It depends on the model reading `README.md`; if no backend/bootstrap/Java target is read, no profile summary is injected.
- It does not implement or verify workflow roles as autonomous agents.
- It does not evaluate Test -> Feat -> Refactor.

## Decision

Keep backend profile summary injection in the v0.3.0 path.

Next useful loop:

1. Create a minimal `.persona/workflow/plan.md` artifact path for the `blackbear` planning role.
2. Keep it backend-only and non-enforcing.
3. Do not implement `Charles`, `jaeki`, or `roach` as autonomous agents yet.
