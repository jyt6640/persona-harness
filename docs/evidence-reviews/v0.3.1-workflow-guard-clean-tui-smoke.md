# v0.3.1 Workflow Guard Clean TUI Smoke

## Goal

Verify whether a clean OpenCode TUI request as short as:

`README 보고 구현해줘`

naturally drives the Persona Harness workflow through:

1. `npx ph workflow guard implement`
2. `npx ph plan --implement`
3. implementation
4. filled implementation report
5. filled review report
6. `npx ph workflow guard final`

This smoke evaluates AI-facing workflow discipline. It is not generated app product-quality certification.

## Aggregate Result

Summary:

- Start workflow discipline: 3/3
- Implementation report filled: 3/3
- Review report filled: 3/3
- Final guard PASS: 1/3
- Evidence present: 3/3

Interpretation:

The current v0.3.1 surface is good at starting implementation through the accepted plan and report templates. It is not yet reliably good at completing final workflow discipline from a short TUI request.

The key failure is not Java Clean Code guidance. The key failure is that the agent can still:

- perform raw shell checks before switching to `npx ph bearshell`;
- record that raw shell usage in reports;
- get stuck with a final guard failure even after later corrected verification.

## Decision

Do not treat the current short TUI workflow as fully stable.

The next improvement should be script-level workflow routing, not more prose in README or plan templates.

Recommended next narrow surface:

- `npx ph workflow start implement`
  - checks accepted plan and report templates;
  - prints the exact implementation sequence;
  - makes `npx ph bearshell` the default verification route;
  - replaces the agent having to infer the sequence from multiple documents.

- `npx ph workflow finish implement`
  - checks filled implementation and review reports;
  - checks evidence;
  - checks command discipline;
  - runs or wraps the existing final guard;
  - reports one concrete next fix when incomplete.

Existing `ph workflow guard implement/final` should remain as lower-level strict gates.

## Limitations

- Only three clean OpenCode TUI runs.

- The generated apps were smoke-observed only as workflow artifacts; this document does not certify generated app quality.
- Runs were local clean-project checks, not public npm registry install checks.
