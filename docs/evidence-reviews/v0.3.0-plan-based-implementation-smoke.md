## Shape Review

Positive signals:

- Gradle only; no Maven files created.
- Domain repository interfaces are in domain packages.
- JDBC repository implementations are in infrastructure packages.
- Application services depend on repository ports, not concrete infrastructure implementations.
- Application services do not own `Map`, `List`, `AtomicLong`, `nextId`, or `idCounter` storage/id sequence state.
- Domain models are classes, not records.
- Domain models hold behavior such as `Equipment.rentOne()`, `Equipment.returnOne()`, and `Rental.returnBy(...)`.
- Request/response DTOs and application command/result DTOs are separated.
- Flyway migration and H2 runtime were used according to the plan.

Weak signals / limitations:

- Domain entities expose `assignId(...)` setters for persistence assignment. This is acceptable for the smoke but should be watched if it becomes widespread.
- `GlobalExceptionHandler` import ordering is a little rough.
- Validation/error response shape is usable but not a final API standard.
- No concurrency control for simultaneous rental attempts.
- H2 was verified; other RDBMS behavior was not.
