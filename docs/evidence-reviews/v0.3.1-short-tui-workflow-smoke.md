## Decision

v0.3.1 short-request workflow is partially positive:

- The main requested behavior works: a very short prompt triggered `ph plan --implement`, `ph workflow check`, `ph bearshell` test/build verification, and implementation report filling.
- Generated backend shape is strong for this run.
- Remaining gap is workflow completion discipline beyond implementation: review report filling is not automatic, and the agent can still read `.persona/rules` directly unless the prompt is made even stricter.

## Follow-up

- Keep `ph workflow check` as the primary status surface.
- Treat review-report filling as an explicit next workflow step, not automatic proof.
- The implementation prompt now reinforces: do not read `.persona/rules` directly; rely on injection summary and accepted plan.
- The implementation gate now also reinforces: before the final answer, fill `.persona/workflow/review-report.md` after manual QA and run `npx ph plan --report-filled review`.
- A future smoke should check whether that stricter wording reduces direct `.persona/rules` reads.
