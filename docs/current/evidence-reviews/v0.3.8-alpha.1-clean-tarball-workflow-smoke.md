# v0.3.8-alpha.1 Clean Tarball Workflow Smoke

## Goal

Verify the current Persona Harness package surface from a clean project before preparing `v0.3.8-alpha.1`.

This smoke checks workflow reliability only. It does not certify generated application product quality.

- Source repo: `/Users/yongtae/Desktop/persona-harness`
- Package under test: local tarball built from the current worktree

## Commands


```markdown

- Register members by email and name.
```

## Observed Result

`npx ph workflow split README.md` created 3 tickets:

- `step-1`: Equipment catalog
- `step-2`: Member registration
- `step-3`: Loan workflow

`npx ph workflow next` surfaced the first pending ticket with:

- ticket id;
- title;
- task-card path;
- archive command.

`npx ph workflow continue` surfaced:

- accepted plan;
- template implementation/review reports;
- no filled continuation evidence;
- pending `step-1` ticket;
- fallback implementation rail;
- read coverage guidance;
- `npx ph bearshell` guidance for long file ranges.

`npx ph workflow finish implement` exited non-zero as expected because implementation was not performed. It reported:

- implementation report must be filled;
- review report must be filled;
- `.persona/evidence` must contain evidence;
- pending tickets remain: `step-1`, `step-2`, `step-3`;
- `npx ph workflow next` and archive guidance for each ticket.

`npx ph workflow check` reported `WARN` as expected and showed pending ticket details plus next/continue guidance.

`npx ph doctor` reported:

- Runtime readiness: `PASS`
- Persona package version: `0.3.8-alpha.0`
- npm registry alpha/latest: `0.3.8-alpha.0`
- OpenCode: present in this local environment
- `.opencode/opencode.json`: present
- `.persona/harness.jsonc`: present
- `.persona/rules`: present
- `.persona/workflow/plan.md`: present
- `.persona/evidence`: missing, expected before implementation
- stale fixture scan: `PASS`

## Decision

PASS for release-prep purposes.

The smoke confirms that clean tarball install and workflow command surfaces behave as expected:

- bootstrap creates the expected harness project structure;
- README step splitting creates tickets;
- next/continue surfaces pending work;
- finish blocks premature completion;
- check remains report-only but warns about pending work;
- doctor reports runtime readiness.

## Known Gaps

- This smoke did not run OpenCode generation.
- This smoke did not verify generated Java/Spring application quality.
- This smoke used a tarball built before the `0.3.8-alpha.1` version bump, because the smoke itself is the evidence for the release-prep bump.
- `.persona/evidence` was missing by design because no OpenCode implementation run was performed.
