# v0.2.0-alpha Support Contract

## Status

`v0.2.0-alpha` is the current support posture for Persona Harness.

The package version is still `0.2.0`; `alpha` describes the support contract, not an npm registry release. Public npm publish, GitHub release, version tag, and release branch remain deferred.

## Supported Install Paths

Supported for this alpha:

- local path install from the Persona Harness repository,
- tarball install through `npm pack`,
- clean project initialization through `npx persona-harness init` or `npx ph init`,
- OpenCode plugin registration generated by init,
- Java/Spring Gradle backend target injection.

Not supported for this alpha:

- `npm install -D persona-harness` from the public registry,
- global install assumptions,
- desktop app packaging,
- frontend/infra productization,
- production support for generated applications.

## Supported Runtime Contract

Persona Harness supports this runtime behavior in `v0.2.0-alpha`:

- Java/Spring `targetFile` detection.
- fileRole-based injection for Java/Spring backend files.
- `.persona/rules` loading from the target project.
- metadata-only evidence under `.persona/evidence/phase0`.
- diagnostics-only rule report through `npm run report:rules`.
- MVP scope check through `npm run check:scope:strict`.
- injection value status check through `npm run check:injection-value`.
- bounded command helper through `npx ph bearshell`.
- injection-block awareness that prefers the Persona Harness `ph` command surface for repo inspection, CLI smoke tests, and large output checks.

## Supported Java/Spring MVP Scope

The release-facing MVP is Java/Spring backend Clean Code injection.

Supported guidance areas:

- Gradle-first build shape.
- Controller/Service/Repository/DTO responsibility boundaries.
- Domain repository interface with infrastructure implementation.
- Application Service orchestration-only behavior.
- Service not directly owning storage state or id sequence.
- Metadata evidence that explains which role/rules were injected.

## Explicit Non-Goals

This support contract does not claim:

- generated Spring app product quality,
- test sufficiency,
- TDD workflow support,
- rule compliance enforcement,
- Guard/AST/linter checks,
- exact package/class naming,
- frontend/infra/desktop support,
- public npm install support,
- OMO native sidecar or tmux/session-context parity for `ph bearshell`.

## Verified Clean Behavior

The clean OpenCode smoke in `docs/current/clean-opencode-ph-bearshell-smoke.md` verified:

- local install init,
- tarball install init,
- installed `ph` bin resolution through `npx`,
- OpenCode Java target injection evidence,
- model use of `npx ph bearshell` for repo/build/test checks,
- no user-facing `omo sparkshell` command in the smoke log.

## User-Facing Command Guidance

Inside a project where Persona Harness is installed as a dev dependency, use:

```bash
npx ph init
npx ph bearshell --shell 'gradle test'
```

`ph bearshell` remains the Persona Harness command surface. `npx ph ...` is the reliable invocation form for local/tarball project installs unless the bin is already on `PATH`.
