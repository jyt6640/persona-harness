# v0.3.0 Workflow Report Status Lifecycle

Status: historical v0.3.0 report-marker snapshot. Do not select this document
as current lifecycle or release guidance. The current fail-closed contract is
[`workflow-closure-state-machine-design.md`](workflow-closure-state-machine-design.md).

## Goal

Keep workflow report status labels honest after a report template has been filled.

The narrow problem came from the live HTTP QA OpenCode smoke: `.persona/workflow/implementation-report.md` was filled with Gradle and HTTP evidence, but still said:

```text
Status: template
```

That made the evidence harder to read even though the report content was useful.

## Decision

Use a minimal explicit marker:

```bash
ph plan --report-filled implementation
ph plan --report-filled review
```

The command rewrites only the selected workflow report `Status:` line:

```text
Status: filled
```

It does not update `.persona/workflow/plan.md`.

## Historical Lifecycle

The v0.3.0 lifecycle was:

```text
template -> filled
```

- `template`: generated by `ph plan`;
- `filled`: user or implementation/review agent has filled the report and explicitly marked it.

`completed` is intentionally not added yet. Completion could imply stronger review or product-quality meaning than the current v0.3.0 workflow supports.

## Scope

This is local workflow evidence metadata.

It is not:

- a build/test gate;
- rule enforcement;
- product-quality certification;
- autonomous agent orchestration;
- a replacement for manual QA evidence.

## Commands

Implementation report:

```bash
ph plan --report-filled implementation
```

Review report:

```bash
ph plan --report-filled review
```

The report templates generated by `ph plan` now remind the implementation/review actor to run the relevant command after filling the file.

## Verification

Covered by:

- `tests/persona-harness-plan.test.ts`

Manual CLI smoke should verify:

```bash
node dist/cli/index.js plan --force
node dist/cli/index.js plan --report-filled implementation
node dist/cli/index.js plan --report-filled review
```

Expected result:

- `.persona/workflow/implementation-report.md` contains `Status: filled`;
- `.persona/workflow/review-report.md` contains `Status: filled`;
- `.persona/workflow/plan.md` keeps its existing acceptance status.
