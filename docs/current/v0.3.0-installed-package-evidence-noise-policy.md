# v0.3.0 Installed Package Evidence Noise Policy

## Goal

Keep clean Java/Spring project evidence focused on the user's generated project, not Persona Harness files installed under `node_modules`.

## Problem

The v0.3.0 plan-based implementation smoke showed useful Java target evidence for generated application files, but it also showed noise:

- OpenCode read Java fixture files under `node_modules/persona-harness/packages/shared-skills/.../fixtures/java/...`.
- Those files were treated as Java targets.
- Metadata evidence was written for vendored reference fixtures.

That made the clean project evidence harder to read. It did not break the generated application, but it weakens later quality/rubric review because installed package internals look like project source.

## Policy

Persona Harness keeps vendored `packages/shared-skills` in the local/tarball package, but installed package internals are not useful clean-project target evidence.

The runtime therefore ignores target files below:

```text
node_modules/persona-harness/
```

This applies to:

- direct read/edit/write target capture;
- Java role discovery from glob/list/find output.

## In Scope

- Suppress injection/evidence for installed Persona Harness package internals.
- Preserve injection/evidence for the user's real Java/Spring files under `src/main/java` and `src/test/java`.
- Preserve vendored shared-skills as packaged reference material.

## Non-Goals

- Do not remove `packages/shared-skills` from the tarball.
- Do not introduce a product-quality gate.
- Do not add AST/linter/Guard enforcement.
- Do not hide evidence for real generated app files.
- Do not claim generated Spring app quality certification.

## Expected Clean Project Evidence

Useful evidence should name project files such as:

- `README.md`
- `build.gradle`
- `src/main/java/.../*Controller.java`
- `src/main/java/.../*Service.java`
- `src/main/java/.../*Repository.java`
- `src/main/java/.../domain/*.java`
- `src/test/java/.../*Test.java`

Evidence should not name:

- `node_modules/persona-harness/packages/shared-skills/.../fixtures/java/...`
- any other `node_modules/persona-harness/...` installed package internal file.

## Verification

Unit coverage:

- direct target capture ignores installed Persona Harness Java fixtures;
- Java role discovery ignores installed Persona Harness Java fixtures while keeping real app Java files.

Manual surface:

- run the hook on an installed fixture target and confirm no `.persona/evidence/phase0` file is written;
- run a mixed glob output and confirm the discovery block/evidence contains only real app Java files.


The repeat run should check whether evidence now stays focused on generated project files.
