# v0.3.0 Workflow History

## Goal

Add a minimal local history surface for workflow artifacts after a run has been completed and used.

This closes the current artifact loop:

```text
ph plan -> fill/use workflow artifacts -> ph history
```

The goal is traceability, not enforcement.

## Command Surface

```bash
npx ph history
npx ph history --id run-001
```

Input:

```text
.persona/workflow/plan.md
.persona/workflow/implementation-report.md
.persona/workflow/review-report.md
```

Output:

```text
.persona/workflow/history/<archive-id>/
├── plan.md
├── implementation-report.md
├── review-report.md
└── summary.md
```

Missing workflow files are recorded in `summary.md`; present files are copied. Active workflow files are not deleted.

## Intended Use

Run `ph history` after:

- the plan has been filled or accepted;
- implementation/report artifacts have been completed or reviewed;
- the run has been used as evidence for the current loop.

Use `--id` when a stable archive name is useful for a review or smoke:

```bash
npx ph history --id equipment-rental-2026-06-20
```

Without `--id`, the archive id is a UTC timestamp.

## Non-Goals

- no workflow engine;
- no autonomous `Charles`, `jaeki`, or `roach` agents;
- no deletion or rotation of active workflow files;
- no product-quality certification;
- no rule compliance enforcement;
- no frontend/infra workflow.
