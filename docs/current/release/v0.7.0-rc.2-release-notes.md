## Summary

`0.7.0-rc.2` prepares the P1 default-path surface after the published
`0.7.0-rc.1` baseline. Persona Harness remains an AI coding workflow rail,
evidence, and continuation harness. This candidate does not certify a
generated application, general CI outcome, or platform-wide reliability.

## Included P1 Scope

- P1 STEP 1 records a full HarnessConfig/default/path audit and retained
  dead-flag inventory. It does not change a default or activate a
  measured-negative feature.
- P1 STEP 2 adds project-local `ph doctor` reachability observation for
  managed AGENTS material. Doctor remains inspection-only and does not
  migrate, delete, or repair observed material.
- P1 STEP 3 adds explicit transactional `ph attach` for existing
  Java/Spring/Gradle projects. It writes only the recognized managed V1
  project-local surface after confirmation, enables PH-run verification, keeps
  `runtimeInjection`, `systemConstitution`, `idleContinuation`, and the Ralph
  loop false, preserves corrupt or unrecognized material, and allows repair
  only for a recognized weak installation.
- P1 STEP 4 adds default-off `features.entrySteering` on the existing OpenCode
  transform surface. It selects the latest user-message session, evaluates
  only that session's first user message, and records bounded status without
  raw prompt or session values. The source-only 32-record bilingual corpus is
  a deterministic corpus observation, not a default-on or efficacy result.
- P1 STEP 5 documents current platform and host boundaries: macOS/Linux with
  OpenCode is the measured surface; Windows is unverified with no support
  claim; a Codex adapter is planned without current product evidence.
