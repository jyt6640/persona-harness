## Summary

`0.7.0-rc.1` publishes the accepted post-`0.6.0` workflow and package surface
for next-channel review. Persona Harness remains an AI coding workflow rail,
evidence, and continuation harness. This release candidate does not certify a
generated application or general CI outcome.

## Included Accepted Scope

- Host-neutral `ph go` creates a deterministic prepared-project entry with
  transactional recovery limited to cooperative local writers and ordinary
  workspace edits.
- Plaintext finish now requires substantive implementation/review report
  evidence before report-filled transition, renders one truthful next action
  plus at most one phased command, and keeps loop plaintext/JSON prompt parity.
  `workflow closure next --json` remains the structured companion; there is no
  `finish --json` contract.
- Root public discovery is narrowed to the human front door while legacy direct
  paths remain operational. Developer aliases retain advanced discovery, and
  invalid smoke/feedback writer arguments fail without report or `.persona`
  mutation.
- The npm package no longer distributes the 28 diff-rules assets. The
  repository corpus and current classification/delivery records remain.
- Public entry docs, QUICK-DEMO, and the advanced-surface index clarify the
  Java-first package boundary. Dormant non-Java references remain source-only;
  Role Checklist Relay remains an advanced preview, not automatic orchestration.
- The Java Iron List matrix is documentation only. Three initial
  high-precision Java conventions (`java.raw-type`, `java.optional-get`, and
  `java.mutable-static`) are warn-only and nonblocking. The accepted attendance
  observation recorded zero false positives for its bounded observation scope;
  it is not a general precision or enforcement claim.
- `ph workflow finish implement --reverify` and explicit
  `--reverify --ci` provide the Item 19 POSIX Java/Spring/Gradle fixed-argv,
  no-shell reverification surface. It uses bounded, structurally redacted
  `ph-ci-reverification.1` artifacts; Windows remains unavailable.
- The Item 25 PH-core/language-pack/adapter boundary is design-only. It does
  not move modules, introduce adapters, or implement another language pack.

## Supported Boundaries

- `runtimeInjection=false` remains the default and no hook is required.
- Plaintext finish exit behavior remains the CI authority; closure JSON remains
  a companion diagnostic artifact.
- CI reverification is limited to the accepted POSIX Java/Spring/Gradle catalog,
  fixed `./gradlew test` then `./gradlew build` argv, and its documented
  identity/mutation/status rules.

## Parked Or Not Accepted

- Item 20 hook work remains PARK.
- Item 23 warn-to-block promotion is NO-GO.
- Item 27 non-Java language packs remain PARK.
- No `finish --json` or runtime/default/schema/evidence-schema movement is
  included. The release moves only npm `next`; `latest` remains `0.6.0`.
