## Version

`v0.4.0-rc.8`

`next`

`0.4.0-rc.8` is a next-channel prerelease candidate for the opt-in TDD Workflow
Rail. `latest` must not move.

Before publish, registry `persona-harness@next` still points at
`0.4.0-rc.7` with gitHead
`640b8d3833e8de12657cdebf4ff0bc2877878c6d`.

## Summary

This candidate packages the QA/External-accepted TDD Workflow Rail:

- `enforce.tdd` remains default-off;
- `ph workflow test` records red evidence only when strict
  `enforce.executeVerification=true` lets PH directly run Gradle/JUnit and
  observe a testcase `<failure>`;
- strict-off TDD is advisory/unavailable and writes no fake red/green evidence;
- compile/no-JUnit failure and JUnit `<error>` do not count as valid red;
- workflow closure can block archive/finish with `tdd-red-evidence-missing` or
  `tdd-not-red-then-green`;
- `ph workflow check`, `ph workflow archive <ticket>`, and
  `ph workflow finish implement` can record green evidence for the same
  ticket/test id after red evidence.
