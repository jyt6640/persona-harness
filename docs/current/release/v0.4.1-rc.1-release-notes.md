# v0.4.1-rc.1 Release Notes

`v0.4.1-rc.1`

`next`

This is release prep for the next-channel package on the `0.4.x` line. Registry
`latest` remains official `0.4.0`, and `alpha` remains `0.3.9-alpha.8`.

Registry `next` briefly moved to a superseded `0.5.0-rc.1` build. That package

## Summary

`0.4.1-rc.1` packages post-0.4.0 accepted local/current surfaces:

- instruction inference/adoption/check preview;
- read-only A/B evidence reporting;
- precise Spring `bootJar` disabled conformance blocker;
- TDD rail adversarial evidence hardening;
- CodeGraph default downgrade to remote-only developer MCP defaults with
  explicit `--codegraph-preview` opt-in;
- Trusted Publisher workflow hardening for Node 24/npm provenance.

## Instruction Preview

The instruction preview adds:

- `ph instructions infer backend [--json]`;
- `ph instructions adopt [--min-confidence high|medium|low] [--json]`;
- `ph instructions check [--json]`.

Inference writes local `.persona/instructions/inferred.json` and
`conflicts.json` with source references and provenance. Adoption writes a local
review file, `.persona/instructions/adopted.json`, for non-conflict inferred
candidates. Check reads adopted rules only and currently supports the
deterministic Controller/Service/Repository drift check.

This preview does not mutate harness config, workflow closure state, or finish
blockers. It is not company instruction compliance, an app-quality guarantee,
Clean Code superiority proof, a broad linter, or a closure guarantee.

## Evidence Reporting

`ph evidence ab-report [--json]` aggregates local structured
`persona-ab-measurement.1` evidence into `evidence-ab-report.1`. It reports
finish outcomes, blocked-invalid-completion counts, elapsed time,
provider-token totals, read chars, tool calls, and MCP calls when those fields
are present. Missing telemetry remains unavailable rather than fabricated.

This is not token-saving, provider-token saving, product-efficacy,
navigation-benefit, app-quality, or test-sufficiency evidence.

## Deterministic Gates

The release candidate includes a precise `spring.bootjar-enabled` conformance
blocker for executable Java/Spring/Gradle application profiles. It blocks when
`@SpringBootApplication` and a Spring Boot build signal are present and Gradle
disables `bootJar`. It emits file:line evidence and a fix path through the
existing closure/archive/finish guard.

The TDD rail also has additional adversarial tests for stale red evidence with
mismatched JUnit snapshot digests and red evidence under the wrong ticket.

These are scoped deterministic blockers and tests, not a broad AST/linter
product, generated-app certification, full TDD framework, test-sufficiency
proof, broad reliability claim, or closure-success guarantee.

## Developer MCP Defaults

Backend bootstrap now registers only remote `grep_app` and remote `context7` by
default. The local PH CodeGraph wrapper is opt-in:

```sh
npx ph bootstrap backend --codegraph-preview
```

`--no-codegraph` remains accepted as a compatibility/no-op guard for the
default remote-only developer MCP bundle. `--no-developer-mcp` still disables
developer MCP entries. PH does not run `codegraph init` and does not create
`.codegraph/`.

The CodeGraph wrapper metadata is aligned with this behavior:
`registeredWithOpenCodeByDefault=false`, `optInFlag=--codegraph-preview`, and
`tokenSavingsClaimed=false`.

This is not token-saving, provider-token saving, product-efficacy,
navigation-benefit, default-CodeGraph-effectiveness, PH-owned CodeGraph,
CodeGraph replacement, OMO parity, Codex support, broad reliability, or closure
guarantee evidence.
