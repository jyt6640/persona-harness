## Stage 17 Workflow Loop

`ph workflow loop` is an explicit capped fresh-session blocker loop command. It
reads deterministic PH finish/closure blockers, builds a minimal
blocker-depth prompt, runs a fresh `opencode run` session per iteration,
persists `.persona/workflow/workflow-loop-state.json`, and terminates on finish
PASS, no blockers, or iteration cap.

The Stage 17 External local-current smoke accepted command/package viability
only. It did not prove completion-integrity movement, default change, product
efficacy, token saving, app quality, broad reliability, or autonomous
completion.

## Stage 18 Completion-Integrity Measurement

The accepted Stage 18 main rerun is fixture-scoped completion-integrity
evidence for a finish-reachable Java/Spring/Gradle fixture:

| Condition | Final finish PASS | Paired comparison |
| --- | ---: | --- |
| OFF | `0/10` | baseline |
| Internal tool-output trigger | `10/10` | one-sided `p=0.0009765625` |
| External `ph workflow loop` | `7/10` | one-sided `p=0.0078125`; `3/10` cap hits |

Internal met the preregistered positive completion-integrity criterion for this
fixture. External also met the finish PASS comparison criterion, but is
cap-risky because `3/10` external rows hit iteration cap and did not finish.
No default change is made by this measurement.
