## Summary

`0.6.0-rc.1` prepares the post-0.5 Stage 1-13 line for prerelease testers.

The product center remains gate-first:

- workflow/check/finish/archive gates;
- PH-generated evidence and explicit verification records;
- deterministic conformance blockers;
- read-only evidence and P-minus decision-support reports.

Runtime and multi-agent guidance remain preview/opt-in surfaces. This release
candidate does not claim token saving, provider-token saving, product efficacy,
navigation benefit, app quality, full-TDD sufficiency, broad reliability,
closure guarantee, autonomous completion, generated-app certification,
deterministic role enforcement, or automatic completion/downgrade/removal.

## Runtime Injection And Session Boundaries

`runtimeInjection` remains a parked opt-in preview. There is no default change
in this release prep.

Accepted Stage 1-2 work tightened session classification and subagent
boundaries:

- runtime injection and continuation surfaces classify sessions using
  OpenCode session lifecycle data;
- classified main sessions can receive enabled runtime surfaces;
- classified subagent and unknown/unclassified sessions fail closed;
- skip evidence is aggregated rather than emitted per message;
- public multi-agent role names are `test-writer`, `implementer`, and
  `reviewer`.

Legacy role names remain compatibility/readback artifacts only. They are not
active public role keys.

## Continuation And Ralph-loop

Stage 4 unified continuation prompt/gate behavior across workflow continue,
closure, idle continuation, and ralph-loop preview/runtime paths.

Ralph-loop is still default-off. Stage 5 and Stage 8 introduced a retry-capped
blocker continuation runtime path with persisted state and separate
per-blocker/per-session budgets. The CLI preview reports dry-run state with
`workflow-ralph-loop.3`.

Stage 12 is a negative/partial measurement that must remain visible:

- the ON pilot did not exercise ralph-loop;
- ralph-loop remains parked/default-off preview;
- there is no default-ON evidence, autonomous completion proof, closure-success
  guarantee, or broad reliability evidence.

## Role Boundary And Relay Guidance

`workflow role-boundary` remains report-only.

Stage 6 added the report surface with block mode unavailable because stable
per-session role identity is not available at PH runtime boundaries. Stage 10
added heuristic time-window write observation under
`.persona/evidence/role-boundary/`. Those findings are warning/report-only
observations, not deterministic enforcement, closure blockers, auto-fixes, or
blocked writes.

Stage 11 added static relay guidance behind:

```sh
npx ph bootstrap backend --multi-agent-preview
```

The guidance names the current roles and points agents at relay/closure
commands. It is static guidance only. Stage 13 was a partial `n=1` proxy trial:
the static guidance path was observed, but reliable OpenCode role subagent
invocation or orchestration was not observed.
