# v0.3.0 Intake Transcript Fixture

## Goal

Fix the expected `ph intake --interactive` conversation used by the interactive CLI.

This fixture is the transcript and output contract for the implementation.

## Scope

- backend-only project intake;
- planning context only;
- no rule enforcement;
- no product-quality certification;
- no frontend/infra/deployment questions;
- no philosophy/policy overlay questions.

## Happy Path Transcript

Command:

```bash
npx ph intake --interactive
```

## Conditional Transcript: No Persistence

If storage does not need persistence, migration should not be asked.

```text
5. 데이터 저장 방식은 어떤 쪽인가요?
선택 [추천: database]: 1

6. 영속성 기술은 어떤 쪽을 원하나요?
선택 [추천: jdbc-template]: 1

7. 패키지를 무엇 기준으로 나눌까요?
...
```

Expected result:

- `storage: none`;
- `persistence-technology: not-needed`;
- no `migration-style` question, or `migration-style: not-needed` if implementation needs a stable placeholder;
- no partial profile is written until the required flow completes.

## Abort Transcript

If the user exits before completion:

```text
1. 이 프로젝트는 어떤 맥락에 가깝나요?
선택 [추천: solo]: 2

2. 이 프로젝트의 목적은 어느 쪽에 가깝나요?
선택 [추천: production-service]: ^C

Interactive intake aborted. No project profile was written.
```

Expected result:

- `.persona/project-profile.jsonc` is not created;
- existing profiles are not modified;
- no partial profile is written.

## Non-TTY Transcript

```text
Interactive intake requires a TTY. Use `npx ph intake` to create an editable draft.
```

Expected result:

- exit status is non-zero;
- no profile is written.

## Implementation Criteria

`ph intake --interactive` should satisfy this fixture.

Minimum test cases:

- numeric choice writes the matching answer;
- `Enter`, `추천`, and `recommend` write the recommended stable value;
- `미정` writes `undecided` only where supported;
- migration is conditional;
- optional project note is saved to `notes.project`;
- abort writes no partial profile;
- non-TTY fails safely;
- `ph plan` reads the saved profile summary.
