# v0.3.0 Plan Acceptance

## Goal

Add a minimal acceptance state surface for the `blackbear` plan artifact.

The goal is to leave an explicit pre-implementation signal that the architecture/technology plan is still a draft, needs revision, or has been accepted.

## Command Surface

```bash
npx ph plan
npx ph plan --status
npx ph plan --accept
npx ph plan --revise
```

## Status Model

`ph plan` creates `.persona/workflow/plan.md` with:

```text
Status: draft
```

`ph plan --status` reads the current status line.

`ph plan --accept` rewrites the status line to:

```text
Status: accepted
```

`ph plan --revise` rewrites the status line to:

```text
Status: needs-revision
```

If the plan artifact does not exist, status commands fail with a clear message and do not create a plan implicitly.

## Framing

This is a local workflow marker, not an enforcement gate.

The status is meant to help OpenCode and the user see whether implementation is supposed to begin from an accepted plan. It does not block commands, fail builds, certify generated app quality, or prove that the plan is correct.

## Non-Goals

- no autonomous `blackbear` / `Charles` / `jaeki` / `roach` agents;
- no implementation gate;
- no rule compliance enforcement;
- no generated app product-quality certification;
- no frontend/infra workflow;
- no TDD workflow enforcement;
- no AST/linter/Guard.

## Verification

Covered by:

- `tests/persona-harness-plan.test.ts`

Manual CLI smoke:

```bash
npm run build
node dist/cli/index.js plan
node dist/cli/index.js plan --status
node dist/cli/index.js plan --accept
node dist/cli/index.js plan --status
node dist/cli/index.js plan --revise
node dist/cli/index.js plan --status
```

Expected result:

- initial status is `draft`;
- accepted status is persisted in `.persona/workflow/plan.md`;
- needs-revision status is persisted in `.persona/workflow/plan.md`;
- missing plan status commands return `No workflow plan found. Run ph plan first.`

## Next Loop

Run one clean workflow smoke where OpenCode reads `README.md` and `.persona/workflow/plan.md`, confirms or revises the plan status, and only then proceeds to implementation.

Do not add autonomous workflow agents until the status marker proves useful in repeated clean runs.
