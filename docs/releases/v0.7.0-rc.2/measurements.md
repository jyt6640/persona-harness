# v0.7.0-rc.2 Measurement And Observation Summary

This summary records scoped P1 observations carried by the published candidate.
It does not turn them into release-wide quality, reliability, or support
claims.

## Default And Attach Boundaries

The P1 default audit records `features.entrySteering=false` and
`features.runtimeInjection=false` by default. The accepted attach contract
enables PH-run verification only after explicit confirmation; it retains the
measured-negative runtime surfaces as false. This is a configuration and
transaction contract, not a claim about generated application outcomes.

## Entry-Steering Corpus

The source-only bilingual entry-intent corpus contains 16 positive and 16
negative records. Its deterministic current result is 16 true positives, 16
true negatives, zero false positives, and zero false negatives under its
preregistered precision/recall/weighted-error thresholds. False negatives cost
2 and false positives cost 1.

This is a corpus-only detector observation. It does not authorize a default-on
setting or claim product efficacy, broad reliability, or user outcome benefit.

## Platform Boundary

Current host-adapter and product evidence is limited to macOS/Linux with
OpenCode. Windows device/inode and stale-lock/concurrency conclusions are
unmeasured; a Codex adapter has no current product evidence. These are
support-boundary statements, not platform certification results.

## Claim Boundary

Publication does not add a new LEAN, rail-entry, ralph-loop, or
generated-application measurement. All observations remain tied to their original
accepted scopes and do not certify application quality, security, broad
enforcement, or multi-platform support.
